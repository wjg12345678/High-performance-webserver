定时器处理非活动连接
===============

当前版本使用 `timerfd + 最小堆` 管理连接超时，不再使用 `alarm + SIGALRM + pipe + 升序链表`。

> * 每个 `SubReactor` 持有独立的 `timerfd`
> * 定时器容器为最小堆，堆顶永远是最近过期连接
> * `epoll` 直接监听 `timerfd`，统一到 I/O 事件循环
> * 超时回调通过 `http_conn::close_conn()` 收口连接关闭逻辑

实现要点
--------

- 新连接建立后，会为该连接创建 `util_timer`
- 定时器节点保存在 `timer_heap` 中，并记录当前版本号
- 读写活动发生时，连接会延后过期时间并在堆中调整位置
- `timerfd` 触发后，`SubReactor` 读取超时事件并批量清理已过期连接
- 连接关闭时会同时失效对应的定时器元数据，避免 fd 复用后旧定时器误伤新连接
